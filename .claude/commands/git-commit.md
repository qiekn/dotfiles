Commit the current changes following the git workflow rules.

## Instructions

1. Run `git status` to see changes. If there are no changes, tell the user and stop.
2. Run `git diff` (staged + unstaged) and `git log --oneline -5` to understand changes and commit style.
3. Stage the relevant files (prefer specific files over `git add -A`).
4. Craft a commit message following the format in `.claude/rules/git-workflow.md`.
5. Commit and confirm.
